# ADR-0007 — Rust edition 2024, MSRV 1.85

- **Status**: Superseded by ADR-0042
- **Date**: 2026-05-20
- **Decider**: Florentin Dubois
- **Tags**: toolchain, msrv

## Context

The workspace is new — no historical compat constraints. Pinning a low MSRV
(e.g. 1.75) would buy compatibility with old toolchains nobody uses anymore,
at the cost of foregoing genuinely useful new features.

Rust 1.85 (the edition 2024 anchor) brings:
- The `let chains` stabilisation (cleans up nested `if let`s in the state
  machines).
- The `unsafe extern "C"` migration we need to track upstream.
- Edition 2024's `use` semantics for nested mods (cleaner re-exports).

There's no obvious benefit to a lower MSRV — Clever Cloud and any other early
adopter is on a recent toolchain.

## Decision

- `edition = "2024"` on every crate in the workspace.
- `rust-version = "1.85"` (declared in `[workspace.package]` and inherited by
  every member crate via `rust-version.workspace = true`).
- `rust-toolchain.toml` pins **stable 1.85** as the default channel.
- The `fmt` job in CI uses **nightly** because `cargo +nightly fmt`
  understands a few unstable rustfmt knobs we like (`imports_granularity`,
  `group_imports`). This is the only nightly use in the project.

## Consequences

- Bumping the MSRV is a workspace-wide change; do it deliberately, ideally
  pegged to a real new-feature need.
- Contributors must install Rust ≥1.85; the `rust-toolchain.toml` does this
  automatically for anyone using `rustup`.
- CI matrix runs `stable` + `beta` to catch toolchain regressions early.
